# ブログ記事の新規作成

トピック: $ARGUMENTS

## ステップ 1: 記事情報の決定

ユーザーの指定したトピックをもとに、以下を決定する：

- **title**: 記事タイトル（日本語）
- **slug**: URL用のスラッグ（英語、ケバブケース）
- **category**: カテゴリ名
- **tags**: 関連タグの配列
- **description**: 記事の説明文（OGP用、1-2文）

決定した内容をユーザーに提示し、**確認を取ってから**次に進む。

## ステップ 2: 骨子（アウトライン）の作成

記事の全体構成を骨子として提示する：

- 各セクションの見出し（##, ###）
- 各セクションで扱う要点を箇条書きで記載
- セクションの順序と流れの意図を簡潔に説明

例：

```
## はじめに
- 背景と動機
- この記事で学べること

## セットアップ
- 必要な環境
- インストール手順

## 実装
- 基本的な使い方
- ポイントとなるコード例

## まとめ
- 振り返りと所感
```

骨子をユーザーに提示し、**確認を取ってから**次に進む。
ユーザーから「このセクションは不要」「ここを深掘りして」等のフィードバックがあれば反映する。

## ステップ 3: 記事ファイルの作成

以下の規約に従ってファイルを作成する：

- **ディレクトリ**: `content/posts/YYYY-MM-DD-<slug>/`（日付は今日の日付）
- **ファイル**: `index.md`
- **日時**: `created_at` と `updated_at` は現在日時（ISO 8601形式、例: `2026-03-15T00:00:00.000Z`）
- **path**: `/<slug>` 形式

Frontmatter テンプレート：

```yaml
---
title: "タイトル"
created_at: "YYYY-MM-DDTHH:mm:ss.000Z"
updated_at: "YYYY-MM-DDTHH:mm:ss.000Z"
path: "/slug"
description: "説明文"
category: "カテゴリ"
tags:
  - tag1
  - tag2
---
```

frontmatter のみ書き込み、本文はまだ書かない。

## ステップ 4: セクションごとの執筆

骨子の各セクションを**1つずつ順に執筆**する。

各セクションについて：
1. 骨子の要点に沿って本文を書く
2. 書いた内容をユーザーに提示する
3. **ユーザーの確認・フィードバックを受けてから**次のセクションに進む

執筆ルール：
- 日本語の技術ブログ記事として執筆する
- 読者は日本語話者のエンジニアを想定
- コード例がある場合はシンタックスハイライト用に言語を指定する（```js, ```ts など）
- 簡潔で読みやすい文体にする
- 画像を使う場合は同じディレクトリに配置し `![alt](./filename.jpg)` で参照する

## ステップ 5: テキスト校正

全セクションの執筆が完了したら以下を順に実行する：

1. `pnpm lint:text` でチェック
2. エラーがあれば `pnpm lint:textfix` で自動修正
3. 自動修正後もエラーが残る場合は手動で修正
4. 最終的に `pnpm lint:text` がエラーなしで通ることを確認

## ステップ 6: 完了報告

以下の情報をユーザーに報告する：

- 作成したファイルのパス
- 記事のURL path
- lint の結果（pass/fail）
- 記事の概要
