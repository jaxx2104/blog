# Biome Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace ESLint and Prettier with `@biomejs/biome` as a full drop-in toolchain for this Next.js blog, delivered as a single pull request.

**Architecture:** Run Biome's migrate commands to seed `biome.json` from the existing `.eslintrc.json` and `.prettierrc.js` files. Reconcile the generated config with the target policy from the spec. Apply formatting, lint, and import-sort fixes in one mechanical pass. Remove the legacy ESLint/Prettier toolchain (configs, dependencies, the now-obsolete `babel-plugin-styled-components`). Update `package.json` scripts and the GitHub Actions workflow to call `biome` instead of `eslint`. Verify locally with the existing `tsc`, `next build`, and `next dev` flows.

**Tech Stack:** `@biomejs/biome` 2.2.x, pnpm 9.15.9, Node.js 22.21.1, Next.js 15.5, GitHub Actions.

**Spec:** `docs/superpowers/specs/2026-05-01-biome-migration-design.md`

**Working branch:** `docs/biome-migration-spec` (already created from `origin/master`; the spec commit `6fe676a` is the first commit). All tasks below add commits onto this branch.

## Task 2: Reconcile `biome.json` with the target policy

**Files:**
- Modify: `biome.json`

The seeded config from Task 1 may be missing `vcs`, `files.includes`, `assist.actions.source.organizeImports`, or have different formatter defaults. Replace the file with the target policy verbatim.

- [ ] **Step 1: Overwrite `biome.json` with the target policy**

Replace the entire contents of `biome.json` with:

```jsonc
{
  "$schema": "https://biomejs.dev/schemas/2.2.4/schema.json",
  "vcs": {
    "enabled": true,
    "clientKind": "git",
    "useIgnoreFile": true
  },
  "files": {
    "includes": [
      "app/**",
      "components/**",
      "lib/**",
      "styles/**",
      "types/**",
      "**/*.json"
    ]
  },
  "formatter": {
    "enabled": true,
    "indentStyle": "space",
    "indentWidth": 2
  },
  "javascript": {
    "formatter": {
      "semicolons": "asNeeded",
      "quoteStyle": "double",
      "trailingCommas": "all"
    }
  },
  "linter": {
    "enabled": true,
    "rules": { "recommended": true }
  },
  "assist": {
    "enabled": true,
    "actions": { "source": { "organizeImports": "on" } }
  }
}
```

If the installed Biome version is not exactly `2.2.4`, update the `$schema` URL to match the installed version (read it back with `pnpm exec biome --version`).

- [ ] **Step 2: Validate the config**

Run:
```bash
pnpm exec biome check --help > /dev/null
pnpm exec biome explain biome.json 2>&1 | head -5 || true
pnpm exec biome check biome.json
```
Expected: the third command exits zero and prints either no diagnostics or a brief summary stating the file is OK. Schema validation errors will show up here; fix them before continuing.

- [ ] **Step 3: Commit the reconciliation**

Run:
```bash
git add biome.json
git commit -m "chore: align biome.json with target policy (vcs, includes, organizeImports)"
```

## Task 3: Apply `biome check` across the codebase

**Files:**
- Modify: any file under `app/`, `components/`, `lib/`, `styles/`, `types/`, plus root-level JSON, that Biome rewrites (formatting, import sort, safe lint fixes).

This task produces the largest diff in the migration. Keep it in its own commit so reviewers can isolate the mechanical reformat.

- [ ] **Step 1: Run `biome check` in write mode**

Run:
```bash
pnpm exec biome check --write .
```
Expected: stdout lists files reformatted and any remaining diagnostics that need manual attention. The exit code is non-zero if anything cannot be auto-fixed (which is normal at this point).

- [ ] **Step 2: Inspect the resulting diff at a glance**

Run:
```bash
git status --short | head -40
git diff --stat | tail -20
```
Expected: many files touched, mostly under the `includes` directories. Confirm nothing under `content/`, `public/`, `out/`, `.next/`, or `node_modules/` was modified.

- [ ] **Step 3: Re-run `biome check` without `--write` to surface remaining diagnostics**

Run:
```bash
pnpm exec biome check .
```
Expected: either exits zero (no remaining diagnostics) or prints a list of diagnostics that could not be auto-fixed. Capture the count and rule IDs.

- [ ] **Step 4: Resolve any remaining diagnostics**

For each remaining diagnostic, choose one of:

1. **Fix the code** if the diagnostic points at a real issue (preferred for `correctness` and `suspicious` rules — e.g. unused variables, hooks dependency arrays).
2. **Disable the rule** in `biome.json` if it is noisy or not useful for this codebase. Add to `linter.rules` like:
    ```jsonc
    "linter": {
      "enabled": true,
      "rules": {
        "recommended": true,
        "style": { "useImportType": "off" }
      }
    }
    ```
    Use the rule's group (e.g. `style`, `complexity`, `suspicious`) as the key, then the camelCase rule name. Names come from the diagnostic output.
3. **Do not** scatter `// biome-ignore` comments to suppress individual sites. Either fix or disable globally.

After each fix, re-run `pnpm exec biome check .` until it exits zero.

- [ ] **Step 5: Confirm `biome ci` is clean**

Run:
```bash
pnpm exec biome ci .
```
Expected: exits zero with no findings. `biome ci` is stricter than `biome check` (it fails on any formatting drift), so this is the canonical CI gate.

- [ ] **Step 6: Commit the formatting and lint sweep**

Run:
```bash
git add -A
git status --short
git commit -m "style: apply biome check across the codebase"
```
Expected: a single commit containing the mechanical reformat. The diff stat will be large but every change is generated by Biome.

## Task 6: Update the CI workflow

**Files:**
- Modify: `.github/workflows/lint.yml`

- [ ] **Step 1: Edit `.github/workflows/lint.yml`**

In the `lint` job's `steps` list, change the final step from `- run: pnpm lint` to `- run: pnpm lint:ci`. The full file should look like:

```yaml
name: Lint
on:
  push:
    branches:
      - master
      - staging
      - develop
  pull_request:
    branches:
      - master
      - staging
      - develop
jobs:
  lint:
    name: lint
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: corepack enable
      - uses: actions/setup-node@v4
        with:
          node-version-file: ./.node-version
          cache: "pnpm"
      - run: pnpm install
      - run: pnpm lint:ci
```

Only the last `run` line changes. Do not modify `test.yml`.

Run:

Expected: a single-line change from `- run: pnpm lint` to `- run: pnpm lint:ci`.
